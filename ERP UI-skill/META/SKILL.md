## 硬性规则 ALWAYS
1. 所有表单提交按钮统一页面/弹窗底部居右，主按钮右、取消重置左。
2. 数据表格操作列固定最右侧，常规按钮不超过3个，超出收进「更多」下拉。
3. 表单必填项统一红色*标识，全局标签对齐方式保持一致。
4. 状态标签（待处理/已完成/驳回）固定规范色值，文字居中、尺寸统一。
5. 全局间距、字号、颜色全部使用Design Token，禁止写死固定色值和像素。
6. 页面采用标准24栅格布局，表单优先单列/双列/三列固定排布。
7. 危险操作（删除/驳回/清空）必须增加二次确认弹窗。
8. 列表筛选栏固定在表格上方，布局、高度、控件样式全局统一。

## 硬性规则 NEVER
1. 禁止使用渐变、花哨光影、异形圆角非标样式。
2. 禁止同时出现4种以上字号、3种以上主色调。
3. 禁止表格无规则列宽、留白过大、空列冗余布局。
4. 禁止按钮文案不统一（确认/确定、关闭/取消混用）。
5. 禁止省略空状态、加载状态、异常错误状态页面。
6. 禁止表单按钮散落页面中部，不固定底部。

## 快速规范速查表
| 模块 | 统一规范 |
|------|----------|
| 页面边距 | 内外边距统一使用 16px/12px/8px 梯度 |
| 文本层级 | 一级标题>二级标题>正文>辅助文字>占位文字 4级固定 |
| 按钮类型 | 主按钮/次按钮/虚线按钮/文字按钮/危险按钮 五类固定 |
| 表格规则 | 表头固定、分页居右、操作列靠右固定 |
| 表单布局 | 标签统一对齐、必填标*、控件间距统一 |
| 弹窗规则 | 左上角关闭、底部按钮栏居右 |

## 用户画像设计约束
1. 所有页面UI布局、表单复杂度、控件尺寸、信息密度，必须严格遵循 /REFERENCES/user-persona.md 各角色使用习惯、设备偏好、操作痛点。
2. 决策者优先做**数据大屏、聚合看板、少录入多可视化**设计；
3. 运营/采购/财务优先**高密度表格、自定义列、批量操作、筛选导出**设计；
4. 仓库/品质现场角色适配**平板友好、大按钮、简化表单、勾选式录入**；
5. 供应商门户坚持**轻量化、极简、少配置、核心信息突出**原则；
6. 严禁脱离角色真实工作场景，做过度花哨、冗余复杂的页面设计。
