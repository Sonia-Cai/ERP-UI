# 表单设计通用规范
## 1 规范概述
### 1.1 目的
统一系统内所有表单视觉样式、布局结构、控件使用、交互逻辑、校验规则，提升页面一致性、降低用户录入成本、减少业务出错率，适配PC端业务表单场景。

### 1.2 适用范围
适用页面：新增表单、编辑表单、查看详情表单、验货/入库/质检等专用业务表单。
适用终端：PC端。

### 1.3 通用原则
1. 业务优先：按**填写顺序+业务关联度**自上而下排布字段；
2. 极简统一：同类型字段样式、间距、文案、交互保持全局一致；
3. 防错优先：强制格式校验、必填显性标注、关键数据限制输入；
4. 适配兼容：PC端侧重高效录入。

## 2 布局与栅格规范
### 2.0 控件宽度规范

| 类型 | 宽度 | Token | 适用场景 |
| :--- | :--- | :--- | :--- |
| 数字输入框 | 96px | `--input-width-number` | **仅用于纯数字输入**（数量、金额、重量、百分比等） |
| 短字段输入框 | 200px | `--input-width-short` | **仅用于表格内嵌控件**（行内状态、数量、货币单位） |
| 中字段输入框 | 240px | `--input-width-medium` | **表单中的主力宽度**，适用日期、SKU、渠道、仓库名称等绝大多数字段 |
| 长字段输入框 | 400px | `--input-width-long` | 备注、描述、地址等需展示较多文字的字段 |

**禁止规则：**
- 短字段（200px）禁止在表格外使用
- 数字输入框（96px）禁止用于非纯数字字段
- 禁止使用上述四档以外的自定义宽度

### 2.1 页面基础间距
#### 2.1.1 编辑类表单（新增、编辑、弹窗录入、业务填制表单）
布局规则：标签与输入控件为上下布局
- 表单页面左右内边距：24px
- 模块分组上下间距：24px
- 单行字段上下间距：24px
- 标签与输入控件间距：12px

#### 2.1.2 抽屉内单列表单的垂直节奏（落地校验）

适用于右侧抽屉等**纵向 flex 布局**、字段自上而下排列的编辑表单（与 4.1 抽屉式表单配套）。

| 间距关系 | 数值 | Token |
| :--- | :--- | :--- |
| 抽屉内容区左右内边距 | 24px | `--spacing-loose` |
| **相邻表单块**之间的垂直间距 | 24px | `--spacing-loose` |
| **父控件底边** → **子层级灰底区顶边**（见第 12 节） | 16px | `--spacing-base` |

**「表单块」定义（用于对齐 24px 间距）：**

1. **触发字段 + 可选子层级**：主触发控件（如 SKU）与其下方按需展开的子层级（如渠道库存）包在同一分组容器内；该整块与下一个表单块之间为 **24px**。
2. **双列/多列一行**：同一行内的多个字段算一块；该行与上一块、下一块之间各 **24px**。
3. **独占一行的字段**：如审批人、备注各占一块；块与块之间 **24px**。

**实现注意：**

- 若使用 `display:flex; flex-direction:column` 实现抽屉主体，除设置块间距外，滚动区域需 `flex:1; min-height:0; overflow-y:auto`，保证底部操作栏贴底且不挤出视区（与 `components.md` 中固定底栏模式一致时可交叉引用）。
- 当 `gap` / `margin` 在复杂嵌套下表现不符合预期时，允许使用**固定高度占位**（如 24px 的空白节点）作为块间距的兜底手段，但**不得替代**正确的 DOM 分组（见 12.2 节）；占位与 margin 二选一即可，同一产品内保持一种策略，避免混用导致双倍间距。
- 抽屉**横向宽度**（随内容最大需求、上限为视口 80%）见 **4.1 抽屉宽度规范**，与本节垂直节奏配合使用。

#### 2.1.3 查看详情表单（只读详情页）— KV 基础布局

适用于**键值对式**只读信息（单据头、基础属性、审批结论等），与 Figma「查看详情表单」画板中**抽屉内摘要区**的排布一致。

**布局规则：标签与展示内容为左右布局（同一行内：左标签 · 右值）**

| 关系 | 数值 | Token |
| :--- | :--- | :--- |
| 表单页面 / 抽屉内容区左右内边距 | **24px** | `--spacing-loose` |
| **模块分组**（卡片或逻辑段）之间上下间距 | **24px** | `--spacing-loose` |
| **相邻两行** KV（上一行值底 → 下一行标签顶） | **16px** | `--spacing-base` |
| **标签与展示内容**（同一行内水平间距） | **12px** | `--spacing-base-tight` |

**标签与值（只读）**

| 元素 | 规则 | Token |
| :--- | :--- | :--- |
| 标签 | **14px**、常规字重；颜色同编辑态标签 | `--font-size-regular` / `--color-text-body` |
| 值 | **14px**、常规字重；主文案 `--color-text-body`，辅助/占位 `--color-text-secondary`；无数据统一 **`—`**（一字线） | — |
| 禁止 | 使用输入框、下拉、开关等**可编辑控件外观**承载只读值（避免与编辑态混淆） | — |

**C. 单元格内「对照 / 同比」展示（只读）**

| 规则 | 说明 |
| :--- | :--- |
| 同一格内多行 | 允许 **主值 + 次值** 两行叠放；次值可用 `--font-size-small` 或 `--color-text-secondary` 区分层级 |

| 涨跌/差异 | 可用箭头图标 + 百分比/差值；颜色用 `--color-success` / `--color-error` / `--color-warning` 语义色，与 `components.md` 状态色一致 |

### 2.2 列排布规则
1. 短字段（姓名、国家、状态、日期等）：支持多列自适应排布。
2. 长字段（SKU、审批人、备注、规格描述）：强制单列独占一行。
**判定依据**：
- 属于业务流程核心筛选条件，如调拨场景「SKU」、RMA 场景「订单号」；
- 下拉选项数量 ≥ 15 个，单独占行可扩大点击区域，减少误触。
3. 按表单载体区分布局形式：
- 页面式表单 / 抽屉表单（**编辑态**）：支持单列/双列/三列布局，统一采用**标签与输入控件上下布局**；
- 弹窗式表单：固定单列布局，统一采用**标签与输入控件左右布局**。
- **查看详情（只读 KV）**：页面或抽屉内键值区采用**标签与展示内容左右布局**，间距见 **2.1.3**；与编辑态上下布局**不得**混用于同一业务场景下的同一区块。

### 2.3 字段分组规则
1. 按业务模块归类划分：基础信息、物料信息、业务参数、质检信息、备注附件等；
2. 同组字段需业务含义紧密关联，禁止跨业务维度字段混编在同一分组；
3. 分组区块之间遵循统一固定间距，通过**分组标题+分割线/卡片容器**做视觉区分；
4. 复杂长业务表单，支持分组默认展开/可折叠收起。

### 2.4 表单版式类型（一级区块形态）

以下三类为系统内表单**一级区块**（相邻业务段）的标准版式，可与 2.1、2.3、第 12 节组合使用；同一表单内宜**择一为主**或按场景混用不超过两种，避免视觉语言碎片化。

| 类型 | 名称 | 说明 | 典型用途 |
| :--- | :--- | :--- | :--- |
| **①** | **普通表单样式** | 仅由标签 + 控件组成的字段纵向排列；一级之间仅靠 **24px** 块间距（`--spacing-loose`）区分，**无**独立分组标题条、**无**全宽分割线、**无**包裹性卡片容器。 | 字段少、单主题、无复杂从属结构的快捷录入。 |
| **②** | **分割线样式** | **一级与一级之间**用**水平分割线**做区隔（量化见下「一级分割线」）；分割线上方为上一段末字段，下方为下一段首字段；若下一段采用「一级 · 有标题」，标题行规范见下「一级标题行」。 | 多段业务含义并列（如「基础信息 / 业务参数 / 备注」），需强节奏切分但无需卡片嵌套。 |
| **③** | **一级带二级卡片样式** | **一级**为外层业务段；其内包含**二级**内容区（常为灰底 `--color-bg-page` + 内层白底 `--color-bg-container` 表格/列表，规则见**第 12 节**）。一级与一级之间仍保持 **24px**。 | SKU 选后展开渠道库存、入库关联明细、可筛选内嵌表等「主字段 + 从属大块」场景。 |

**一级区块的两种标题形态（适用于 ② 分割线样式，以及 ③ 中作为一级段落的抬头；① 通常不配一级标题）：**

| 形态 | 说明 |
| :--- | :--- |
| **一级 · 有标题** | 显式**分组标题**行：左侧装饰条 + 标题文案，量化见下「一级标题行」；标题行与下方**第一个表单字段**间距 **12px**（`--spacing-base-tight`）。 |
| **一级 · 无标题** | **不展示**分组标题行；仅靠 **24px** 间距、分割线（类型②）或二级卡片外轮廓（类型③）暗示分段。用于段意自明、或上一段标题已足够覆盖下一段的紧凑布局。 |

#### 一级分割线（类型②及需段间分割的场景）

| 属性 | 值 | Token / 实现 |
| :--- | :--- | :--- |
| 粗细 | **1px** | `border-top-width: 1px`（或等价 `height: 1px` 占满宽） |
| 颜色 | **`#E4E6EA`** | **`--color-border-default`**（等同 `--color-gray-3`） |
| 宽度 | 表单内容区内**通栏**（与抽屉/页面内容区左右内边距对齐后的可用宽） | — |

#### 一级标题行（一级 · 有标题）

标题行由**左侧装饰条**与**标题文字**横向排列，二者间距 **12px**（`--spacing-base-tight`）。

| 元素 | 规则 | Token / 实现 |
| :--- | :--- | :--- |
| **左侧装饰条** | 宽 **2px**，高 **16px**，填充色 **`#346BFA`** | `width: 2px; height: 16px; background: var(--color-primary);`（`--color-primary` / `--color-brand-6` 为 `#346BFA`） |
| **标题文字** | 字号 **16px**，字重 **Semibold（600）**，颜色 **`#323B4B`** | `font-size: var(--font-size-h6); font-weight: var(--font-weight-semibold); color: #323B4B;`（与 `--color-gray-11` 同色，可与 `color: var(--color-gray-11)` 等价书写） |
| **装饰条 ↔ 标题文案** | **12px** | `gap: var(--spacing-base-tight);`（推荐标题行容器 `display: flex; align-items: center;`） |
| **标题行 ↔ 下方字段** | **12px** | `margin-bottom` / `padding-bottom` + 字段侧 `margin-top` 二选一，避免叠成双倍；与 **`--spacing-base-tight`** 一致 |

**选用建议：**

- 类型 **③** 与第 **12** 节一一对应：「一级」对应触发字段所在表单块，「二级」对应展开后的灰底子区；**禁止**将后续与触发字段无关的主字段误包进二级卡片内（见 12.2）。
- 类型 **②** 的分割线**不得**替代正确的 DOM 分组；分割线为视觉辅助，块间距仍以 2.1.2「表单块」为准。

## 3 表单标签（Label）规范
1. 文字规则：使用标准业务专业术语，禁用口语化表述、无自定义缩写，文案表意清晰无歧义；
2. 文字字号/字重：标签文字统一 14px、常规字重；
3. 对齐排布规则：页面式表单/抽屉表单，标签采用**顶部置顶排布**；弹窗式表单，标签采用**左侧右对齐排布**；
4. 必填标识：必填项统一红色`*`，固定放置在标签左侧，全局位置、样式统一不改动；
5. 标签宽度：设置全局固定标签最大宽度，统一排版规格，避免标签长短不一导致页面错乱；
6. 文案换行要求：禁止标签文字自动换行，文案过长时需精简业务术语，保证标签单行展示。

## 4 表单载体选型规范
系统表单统一采用三种载体（抽屉式、页面式、弹窗式），结合业务场景、页面层级（二级/三级）、字段复杂度选型，禁止随意选用，确保全系统操作逻辑统一、层级清晰。

### 4.1 抽屉式表单（右侧滑出抽屉）
#### 适用场景（核心）
1. 常用表单（**优先选用**）：所有无需作为三级页面的表单，统一采用抽屉式，作为**二级页面**承载；
2. 列表页关联操作（二级页面）：各模块列表页（如库存列表、订单列表）的快捷新增、快捷编辑、关联详情查看，均用抽屉式表单；
3. 中等复杂度表单：字段数量适中、包含基础业务分组，无需新开独立三级页面，避免割裂操作上下文的表单；
4. **含内嵌表格的表单**：表单内含可筛选/可输入的数据表格时，仍优先使用抽屉式；抽屉**不必**拉满上限，以内容舒适展示为准，宽度上限见下。

#### 抽屉宽度规范

**原则（重要）：** 抽屉宽度**不是**固定为视口的 80%，而是由**当前表单内容所需的最大宽度**（最宽字段、双列一行宽度之和、表格列宽与内边距等）决定；同时不得超过**上限**。

| 概念 | 规则 | Token / 实现 |
| :--- | :--- | :--- |
| 宽度上限 | 不得超过视口宽度的 **80%** | `--drawer-width-max`（默认 `80vw`） |
| 实际宽度 | `min(内容所需最大宽度, 上限)` | 推荐 CSS：`width: max-content; max-width: var(--drawer-width-max);` |
| 内容区内边距 | 左右 **24px** | `--spacing-loose` |

**说明：**

- 「内容所需最大宽度」由设计稿/控件宽度规范（如 `--input-width-long`、表格最小可读宽度、双列并排等）共同推导；窄表单应呈现**较窄抽屉**，避免右侧大块留白。
- 若表格或复杂子区域使用 `width: 100%`，需避免与「按内容收缩」形成循环依赖；可通过列宽、`min-width` 或内层 `max-content` 容器等方式，保证**可计算的内容最大宽度**。
- 示例：视口 1440px 时，上限约为 **1152px**；若内容最大仅需 520px，则抽屉宜约 **520px**（仍受上限约束）。

**不推荐：** 将抽屉写死为固定 `80vw` 或与内容无关的固定像素宽（除非该业务已单独约定最小可读宽度）。

#### 配套布局规则
支持单列/双列/三列布局；标签与输入控件采用**顶部置顶排布**，遵循本规范 2.0、2.1.1、2.1.2（抽屉垂直节奏）、2.2 布局规则。

### 4.2 页面式表单（全屏独立页面）
#### 适用场景
1. 需作为**三级页面**的表单：当表单操作需嵌套在二级页面之下，无法通过抽屉/弹窗承载，需独立打开新页面的场景；
2. 核心复杂业务单据：需跨页面联动、分步填写、字段极多（多分组），且作为三级页面存在的表单（如深层级的主数据编辑、复杂质检单详情及编辑）。

#### 禁用场景
常规常用表单（无需三级页面的），禁止选用页面式表单。

#### 配套布局规则
支持单列/双列/三列布局；标签与输入控件采用**顶部置顶排布**，遵循本规范 2.1.1、2.2 布局规则。

### 4.3 弹窗式表单（居中模态弹窗）
#### 适用场景
1. 轻量快捷操作（不分二级/三级页面）：字段少、内容简单，无需复杂录入的辅助表单；
2. 辅助操作表单：筛选配置、备注补充、操作二次确认、简易参数设置，不占用二级/三级页面主体；
3. 短流程操作：操作步骤少，无需多业务分组、无复杂联动的表单。

#### 禁用场景
1. 禁止用于业务常用表单（需作为二级页面的）；
2. 禁止用于需作为三级页面的复杂表单；
3. 禁止用于字段多、包含长文本/多分组、需复杂联动的表单录入。

#### 配套布局规则
固定单列布局；标签与输入控件采用**左侧右对齐、左右布局**，遵循本规范 2.2 布局规则。

## 5 表单控件选用规范
固定场景绑定固定控件，禁止随意混用：
1. 单行输入框：名称、编号、规格、地址短文本；
2. 多行文本域：备注、问题描述、大段文字说明；
3. 下拉/级联下拉：固定枚举选项、业务类型、SKU、渠道、审批人、仓库、物料分类；
4. 单选框：二选一判定类选项；
5. 多选框：标签、多选属性配置；
6. 日期时间选择器：单据日期、质检时间、生效失效时间；
7. 数字输入框：数量、重量、金额、百分比，限制仅数字输入；
8. 勾选框/开关：功能启用、是否判定类字段；
9. 附件上传：质检图片、单据凭证、资料附件。

## 6 控件状态规范
全局统一6种状态样式，色值、边框、文案一致：
- 默认态、聚焦态、输入中态、**只读态**、**禁用态**、报错态

补充规则：
- 只读：可查看、不可编辑，保留布局结构；
- 禁用：置灰不可点击，业务不可操作场景使用。

## 7 录入校验规则
1. 必填校验：带`*`字段提交必校验，为空拦截并提示；
2. 格式校验：手机号、编号、编码、金额统一正则规则；
3. 数值校验：金额/数量限制非负、固定小数位数、最大最小值限制；
4. 字符限制：各字段统一最大输入字符数，超出禁止输入；
5. 错误提示：文案统一、位置固定、不遮挡控件，红色提示；
6. 业务联动校验：字段联动后自动校验数据逻辑冲突。

## 8 辅助信息规范
1. 占位符：仅作输入引导，不替代标签，文案简洁；
2. 单位标注：元、件、kg、% 统一后置固定位置；
3. 帮助说明：复杂字段用小字备注或问号悬浮提示；
4. 系统自动回填：创建人、创建时间、单据编号等自动生成，无需手动填写。

## 9 交互通用规范
1. 键盘Tab/回车：自动跳转下一个输入框；
2. 下拉控件：支持多字段关键词搜索、选择后鼠标悬浮状态清空；
3. 字段联动：选择选项后自动显隐其他字段、自动计算金额/数量；
4. 长表单：支持回到顶部、临时草稿保存；
5. 行内编辑：仅简单字段允许行内编辑。

## 10 操作按钮规范
1. 按钮位置：表单底部固定区域，与内容分割；
2. 主次层级：主按钮（保存/提交）、线框按钮（新增下一条）、次按钮（取消/重置/返回）；
3. 禁用规则：必填未填、数据校验异常时，提交按钮置灰禁用；
4. 弹窗表单：底部按钮靠右排布，简化操作。

## 11 文案话术统一规范
1. 空提示、报错提示、成功提示、弹窗确认话术全局统一；
2. 禁止使用口语化、模糊化文案，全部标准化业务用语；
3. 查看态表单隐藏所有输入控件，以纯文本 / KV 排布展示信息；版式与间距见 **2.1.3**、**2.1.4**、**2.1.5**（主从表 + 密集行高见 2.1.5）。

## 12 子层级内容区规范

当表单中某字段（如 SKU）选择后动态展开关联内容（如渠道库存、物料明细等），该展开区域为**子层级**，视觉上与父字段形成从属关系，样式规则如下：

| 属性 | 值 | Token |
| :--- | :--- | :--- |
| 背景色 | #F3F5F8 | `--color-bg-page` |
| 内边距（上下左右）| 24px | `--spacing-loose` |
| 圆角 | 无（0px）| — |
| 阴影 | 无 | — |
| 与父字段（触发字段）间距 | 16px | `--spacing-base` |
| 与下方下一个表单字段间距 | 24px | `--spacing-loose` |

> 子层级内部的表格、筛选栏等元素背景仍为白色（`--color-bg-container`），依靠灰底背景与外层区分层次。
> **表格表头规则**：当表格处于灰底卡片内时，表头背景改为白色（`--color-bg-container`），不使用默认的 `--color-bg-page`，避免与卡片背景色重叠导致层次混淆。

### 12.2 HTML 结构约束（与间距强相关）

子层级往往包含**多层嵌套**（灰底外包 → 白底内容区 → 表头/表格/分页/行内操作等）。实现时必须保证**每一层开启的容器都在正确层级闭合**，否则会出现「间距 token 已配置但页面无变化」「子层级与后续字段贴在一起」「抽屉底部按钮栏无法固定或滚动异常」等问题。

**必须遵守：**

1. **分组边界清晰**：触发字段（如 SKU）与按需展开的子层级内容，应包在同一个**分组父容器**内（如 `sku-group`）；该父容器必须在子层级**全部** `</div>`（或等价组件）闭合之后再结束。
2. **后续主字段为同级**：子层级之后的常规表单字段（如 To 渠道、审批人、备注）必须与上述分组父容器为**兄弟节点**，禁止因子层级少写闭合标签而被错误包进子层级或分组容器内部。
3. **交付前自查**：在浏览器开发者工具中检查「分组父容器」的直接子节点是否仅包含「触发字段 + 可选子层级」；若发现 To 渠道、审批人等出现在分组容器之下，即为结构错误，需优先修复后再调间距。

**与 2.1.2 的关系：** 垂直间距（16px / 24px）依赖正确的兄弟结构；结构错误时，单独调整 CSS `gap`、`margin` 或占位节点往往**无法**达到设计稿效果。

## 13 通用设计禁忌
1. 禁止字段无分组、杂乱堆砌；
2. 禁止同功能控件样式、大小、间距不统一；
3. 禁止标签表意模糊、缩写随意；
4. 禁止无校验、无必填标识的业务表单上线。
