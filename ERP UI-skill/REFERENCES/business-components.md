# 业务复合组件规范（business-components.md）
> 本文件收录与 ERP 业务场景强绑定的复合组件。基础原子组件（按钮、输入框、标签等）仍遵循 `components.md`。
> 所有样式取值必须引用 `design-tokens.css` 变量，禁止硬编码。

---

## 零、全局通用交互规则

以下规则适用于本文件所有下拉组件（SKUSelect / ChannelSelect / WarehouseSelect / PersonSelect 等）：

| 规则 | 说明 |
| :--- | :--- |
| **搜索框必须** | **所有下拉筛选项面板内必须包含搜索框**，以下两类除外：① 日期选择类（调拨时间、年度、月份等）；② 纯数字 / 固定少量枚举类（如年度下拉、页码等）。其余所有下拉一律带搜索框，不得省略 |
| **悬浮清除图标** | 已选中选项后，鼠标悬浮触发控件时，右侧箭头图标替换为 ✕ 清除图标；点击清除已选值并重置为占位符状态 |
| **搜索框自动聚焦** | 面板展开时搜索框自动聚焦，用户可立即输入 |
| **点击外部关闭** | 点击面板外部区域关闭面板，不触发清除 |
| **搜索实时过滤** | 输入即触发过滤，无需确认 |
| **空结果状态** | 搜索无匹配时显示各组件对应空状态文案（见各组件文案规范）|
| **禁用态** | 控件禁用时不可点击，背景 `--color-bg-disabled`，文字 `--color-text-disabled` |

## 一、搜索下拉选择器（SearchSelect）

### 1.1 适用场景
需要从较长业务列表中快速定位选项，且选项本身携带多维业务信息（如 SKU、审批人、渠道等）。
替代标准 `<select>`，在以下情况必须使用本组件：
- 选项数量 ≥ 15 条
- 选项需展示两行信息（主标签 + 辅助信息）
- 需要关键词实时搜索过滤

### 1.2 结构规范
```
[触发控件]        ← 高度 32px，宽度遵循输入框宽度规范
  └─ [弹出面板]   ← 宽度 = 触发控件宽度，圆角 --radius-medium，阴影 --shadow-3
       ├─ [搜索框]  ← 高度 32px，宽度填满面板，图标后置（搜索图标）
       └─ [选项列表]
            ├─ 单行选项：主文字 14px / --color-text-body
            └─ 双行选项：主文字 14px + 辅助信息 12px / --color-text-secondary
```

### 1.3 尺寸规范
| 属性 | 值 | Token |
| :--- | :--- | :--- |
| 触发控件高度 | 32px | `--control-height-md` |
| 弹出面板最大高度 | 350px | — |
| 选项行高（单行） | 32px | `--control-height-md` |
| 选项行高（双行） | 50px | — |
| 选项内边距 | 左右 8px | `--spacing-tight` |
| 选项间竖向间距 | 0（列表紧凑排列）| — |
| 搜索框与选项列表间距 | 6px | — |
| 面板内边距 | 6px | — |

### 1.4 交互规范
通用交互遵循「零、全局通用交互规则」，以下为 SearchSelect 专属样式规范：
- 已选中项高亮：背景 `--color-primary-bg`，文字 `--color-primary`
- 悬浮未选中项背景：`--color-gray-1`

### 1.5 状态规范
| 状态 | 触发控件边框 | 说明 |
| :--- | :--- | :--- |
| 默认 | `--color-border-default` | — |
| 展开/聚焦 | `--color-primary-focus` + 外发光 | 面板展示 |
| 已选中 | `--color-border-default` | 显示选中值，悬浮显示清除按钮 |
| 禁用 | `--color-border-default` | 背景 `--color-bg-disabled`，不可交互 |
| 错误 | `--color-error` | 校验失败时 |

## 二、SKU 搜索下拉选择器（SKUSelect）

### 2.1 适用场景
**所有涉及 SKU 下拉筛选的字段，统一调用本组件**，不得使用标准 `<select>` 替代。

### 2.2 文案规范
| 位置 | 文案 |
| :--- | :--- |
| 触发控件未选中占位符 | `请选择对应SKU` |
| 搜索框占位符 | `支持搜索：SKU编码、地区、品类、物料编码` |
| 搜索无结果 | `暂无匹配的 SKU` |

### 2.3 选项展示格式（双行）
```
主行：SKU 编码                              14px / --color-text-body / 加粗
辅助行：🏳️ 销售地区 · 品类 · 物料编码 · 产品名称   12px / --color-text-secondary
```

### 2.4 宽度
表单中统一使用 `400px`（对应 Figma 实测宽度，超出标准 4 档，仅 SKU 字段例外）。

### 2.5 搜索规则
支持以下字段关键词实时过滤：
1. SKU 编码
2. 销售地区
3. 品类
4. 物料编码

## 三、渠道树选择器（ChannelSelect）

### 3.1 适用场景
**所有字段名含「渠道」二字的下拉选择，统一调用本组件**，包括但不限于：
- 表单字段：To 渠道、From 渠道
- 筛选栏：From 渠道筛选

### 3.2 文案规范
| 位置 | 文案 |
| :--- | :--- |
| 触发控件未选中占位符 | `请选择对应渠道` |
| 搜索框占位符 | `支持搜索：地区、渠道` |
| 搜索无结果 | `暂无匹配的渠道` |

### 3.3 树形结构
一级分类为固定 5 个，下挂二级渠道选项：

| 一级分类 | 二级渠道 |
| :--- | :--- |
| B2C线上 | AMZ-S、WM-S、TK-S、Kohls-线上-B2C、Temu-线上-B2C、Monti-线上-B2C、Lowes-线上-B2C、GO10-线上-B2C、Wayfair-线上-B2C、Chewy-线上-B2C、Costco-线上-B2C、OTTO、AMZ-DE、Mercado、Shopvidi |
| B2C线下 | 线下(TGT DVS)、Grainger-线下-B2C |
| B2B线上 | AMZ-C、WM-C、TK-C、BBY-C、QVC-C、THD-C、TGT-C(勿用) |
| B2B线下 | — |
| 特殊业务渠道 | DREO、HSN |

> `TGT-C(勿用)` 渠道需在选项文字后附加灰色「勿用」标注，可选但不推荐使用。

### 3.4 结构规范
```
[触发控件]  ← 高度 32px，宽度 240px
  └─ [弹出面板]  ← 宽度 240px，最大高度 350px，圆角 --radius-medium，阴影 --shadow-3
       ├─ [搜索框]  ← 占位符「支持搜索：地区、渠道」，图标后置
       └─ [树形列表]
            ├─ 一级分类标题（箭头图标 16px + 分类名，可折叠/展开）
            │    └─ 二级渠道选项（缩进 20px）
            └─ ...
```

### 3.5 交互规范
- 一级分类默认全部**展开**，点击标题可折叠/展开
- 已展开：面性下箭头；已折叠：面性右箭头（旋转 -90°）
- 图标颜色 `--color-text-secondary`
- 搜索时临时展平为平铺列表（不显示分组），命中项高亮
- 选中逻辑同 SearchSelect 1.4（悬浮清除按钮）

### 3.6 宽度
表单及筛选栏统一使用 `--input-width-medium`（240px）。

## 五、日期选择器扩展规范（DatePicker）

### 5.1 默认值规则
- **新增表单**中的日期字段：**默认填充当前日期**，用户可修改
- **编辑表单**中的日期字段：回显原始记录日期

### 5.2 弹出面板
- 圆角：`--radius-medium`
- 阴影：`--shadow-3`
- 当前日期高亮：背景 `--color-primary`，文字 `--color-text-inverse`

---

## 六、规范约束

1. 以上业务组件的触发控件高度**统一 32px**，与原子输入框保持一致，无例外
2. 弹出面板**不计入表单布局宽度**，允许超出父容器宽度，但不得超出视口
3. 禁止将业务复合组件与标准 `<select>` 混用于同一表单的同类字段
4. 所有组件的交互状态（聚焦/已选/清除/禁用/错误）必须完整实现，不得省略
