# 查看类表单设计通用规范

本规范与 **[`form-standard.md`](./form-standard.md)（表单设计通用规范）** 并列：后者主司**编辑态**录入、控件选用、载体选型及与编辑共用的 Token；**本文件专司只读查看**的版式、信息架构与禁则。色值、间距、字号一律使用 **`ASSETS/design-tokens.css`** 中的变量，禁止随意写死。

---

## 1 规范概述

### 1.1 目的

统一系统内**查看详情**、**只读抽屉**、**主从列表 + 详情**等场景的版式与交互边界，保证只读信息与编辑态**可区分**、阅读路径清晰，并与列表页、数据表、壳层规范一致，降低设计与实现偏差。

### 1.2 适用范围

- **适用页面**：单据/主数据的只读详情、列表行「查看」打开的右侧抽屉、整页只读 + 大表等以**展示**为主的界面。  
- **适用终端**：**PC 端**（与 `form-standard.md` **§1.2** 一致）。  
- **不适用**：本规范**不**规定各业务线的字段清单、必选列或接口契约；业务字段以产品 / 研发定义为准。

### 1.3 与《表单设计通用规范》的关系

| 文档 | 职责 |
| :--- | :--- |
| **`form-standard.md`** | 编辑态布局（**§2.1.1**、**§2.1.2**）、**一级区块版式 §2.4**（与查看类分段**共用**）、**抽屉宽度 §4.1**（编辑与查看**共用**）、控件与校验等。 |
| **本文件（view-form-standard）** | 只读 **KV**、**复合版式**、**只读抽屉**、**主从**、**两种典型抽屉信息架构**、查看类**列排布**等。 |

实施时：**Token、抽屉宽度算法**以 `form-standard.md` **§4.1** 为准；**只读 KV 与查看抽屉内结构**以**本文件**为准。

### 1.4 通用原则

1. **只读可辨**：禁止用输入框、下拉、开关等**可编辑控件外观**承载只读值，避免与编辑态混淆。  
2. **上下阅读**：键值区统一**标签在上、详情在下、左对齐**（与编辑态字段块「上下」节奏一致）；若历史画板为左右 KV，落地仍以本规范为准。  
3. **空态统一**：无数据展示统一为 **`—`**（一字线）。  
4. **底栏统一**：查看类只读抽屉底栏默认为 **取消**（左，次按钮）+ **确定**（右，主按钮）；若业务需「导出 / 打印 / 关闭」等，须**单独约定**，勿与全系默认混用。  
5. **画板与 Demo 定位**：Figma 与静态 HTML 仅提供**布局与数值示例**；**强制条款**以本文件及 `form-standard.md` 为准，勿将某一画板的固定 px 写死为全系常量。

## 2 只读 KV 基础布局

适用于**键值对式**只读信息：单据头、基础属性、审批结论、抽屉内摘要区、灰底卡片内字段等；**不**绑定某一业务画板或字段清单。

**布局规则：标签（标题）在上、展示内容（详情）在下，均左对齐；与编辑态字段块同为「上下」阅读顺序。**

| 关系 | 数值 | Token |
| :--- | :--- | :--- |
| 表单页面 / 抽屉内容区左右内边距 | **24px** | `--spacing-loose` |
| **模块分组**（卡片或逻辑段）之间上下间距 | **24px** | `--spacing-loose` |
| **字段块内：标签底边 → 详情顶边**（垂直间距） | **12px** | `--spacing-base-tight` |
| **相邻字段块**（上一块详情底边 → 下一块标签顶边） | **24px** | `--spacing-loose`（与编辑态单行字段块间距 **§2.1.1** 对齐） |

**标签与值（只读）**

| 元素 | 规则 | Token |
| :--- | :--- | :--- |
| 标签 | **14px**、常规字重；**左对齐**；可与正文区分时使用 **`--color-gray-7`**，或与编辑态一致用 **`--color-text-body`**（产品内统一即可） | `--font-size-regular` |
| 值 | **14px**、常规字重；**左对齐**；主文案 **`--color-text-body`**，辅助 **`--color-text-secondary`**；无数据 **`—`** | — |
| 禁止 | 使用输入框、下拉、开关等**可编辑控件外观**承载只读值 | — |

> **与编辑态对照**：编辑态「标签—控件」**12px**、块与块 **24px**；只读 KV 块内 **12px**、相邻块 **24px**，便于对照记忆。勿与旧版「左右 KV」混用。

## 3 复合版式（整页、表、只读抽屉、主从）

适用于：只读**整页**或只读**抽屉**中，除 **§2** 的 KV 外，还包含**大表格**、**列表 + 详情主从**等组合。下列为**版式原则**；**具体列名、筛选个数、某稿上的绝对像素**不作为全系强制，数值示例见 **`view-form-reference.md`**。

### 3.1 整页只读 + 主内容为大表格

| 元素 | 规则 | 说明 |
| :--- | :--- | :--- |
| 壳层 | 与 `navigation-shell.md` 一致 | 侧栏、顶栏页签、右上状态区等 |
| 主内容区 | 内边距 **24px**（`--spacing-loose`） | 与列表页、编辑页内容区对齐 |
| 表头行高 | 与全系统**数据表**表头统一 | 背景、字色遵循 `components.md`（或后续 `table-standard.md`） |
| 数据行行高 | **富内容**与**紧凑**两档**并存**，按表类型选用 | 推荐像素与画板对照见 **`view-form-reference.md` §4** |
| 横向滚动 | 列数多时可出现横向滚动 | 表体区域 `overflow-x: auto`，避免挤压壳层 |

> **行高分档**：勿将某一 Demo 稿的绝对像素写进全局常量；两档由信息密度决定，**非**全系二选一。

### 3.2 右侧只读抽屉（详情补充）

| 元素 | 规则 | Token / 说明 |
| :--- | :--- | :--- |
| 抽屉标题栏 | 高 **56px** | 与编辑抽屉一致；标题与关闭图标，无保存类主操作 |
| 抽屉宽度 | 不得超过视口 **80%**；`min(内容舒适宽度, 上限)` | 见 `form-standard.md` **§4.1**；**不得**写死为某一固定 px |
| 抽屉内多列摘要 | 宜与 **`--input-width-medium`** 栅格倍数协调 | 「上标题、下值」；标题底到内容顶约 **12px**（`--spacing-base-tight`） |
| **一级分组标题** | 与编辑态 **共用 `form-standard.md` §2.4「一级标题行」** | 2×16 主色条 + 16 Semibold 等 |
| **段间分割线** | 与 **§2.4「一级分割线」** 一致 | **1px** / `--color-border-default` |
| **抽屉底栏** | 高 **56px**；按钮高 **32px**（`--control-height-md`） | 默认 **取消** + **确定**（见 **§1.4**） |

### 3.3 单元格内「对照 / 同比」展示（只读）

| 规则 | 说明 |
| :--- | :--- |
| 同一格内多行 | 允许 **主值 + 次值** 两行叠放；次值可用 `--font-size-small` 或 `--color-text-secondary` |
| 对照连接符 | 固定 **`VS`**（或产品统一中文「对比」），勿混用多种符号 |
| 涨跌/差异 | 箭头 + 百分比/差值；颜色用 **`--color-success` / `--color-error` / `--color-warning`**，与 `components.md` 一致 |

### 3.4 主从结构（主区列表或表 + 只读抽屉）

- 主区可出现**筛选行、表格工具栏、可展开行**等：与列表页统一部分见 **`page-list.md`**；表格细则待 **`table-standard.md`**。  
- 点选主行打开右侧**只读抽屉**；抽屉内遵守 **§2**、**§3.2** 及 **`form-standard.md` §4.1**。  
- 「主从 + 紧凑表」一稿的像素与 Figma 链接见 **`view-form-reference.md` §3**。

## 4 两种典型抽屉落地（形态一 / 形态二）

主从场景下，右侧只读抽屉内常见**两种复杂度**。二者均遵守 **§2**、**§3.2**、**`form-standard.md` §4.1**；差异在是否引入「**摘要区 + 一级分段 + 二级灰底卡片**」。

| 维度 | **形态一：抽屉内连续只读 KV** | **形态二：摘要 + 一级分段 + 二级卡片** |
| :--- | :--- | :--- |
| **适用** | 字段量适中、单主题单据头式信息 | 多维度分段（如外箱 / 包装 / 产品），段内字段成簇 |
| **抽屉主体** | **单列栈**为主；短字段可 **双列一行**（每格内仍为上下 KV） | **摘要区**（常三列×多行）→ **一级标题行**分段 → **`--color-bg-page`** **二级卡片**（内边距 **24px**；卡片内纵向块间距可与画板对齐 **16px**，列间距 **24px**） |
| **一级标题 / 弱链** | 一般无需多段；必要时单段可用 **`form-standard.md` §2.4** | 每段前 **2×16 条 + 16 Semibold**；标题旁可有弱链（如「填写示例」） |
| **与 §2.4 关系** | 以 **类型①** 块间距为主 | 与 **类型③「一级带二级卡片」** 视觉同构；二级区为**只读 KV**，禁止可编辑控件外观 |
| **静态 Demo** | `case-studies/good-case.md` **§4** → `demos/channel-transfer-view.html` | `case-studies/good-case.md` **§5** → `demos/sku-packaging-dimensions-detail-view.html` |
| **画板索引** | `view-form-reference.md` **§2**（画板一） | `view-form-reference.md` **§3**（画板二，含上下布局稿） |

**选型建议：** 字段少、无强分段语义 → 形态一；**多业务段、段内成簇** → 形态二。同一产品内形态不宜过多。

## 5 列与字段排布（查看类）

与 `form-standard.md` **§2.2** 中编辑态「短/长字段」判定**同源**：查看抽屉内的 KV 排布同样遵循——

1. **短字段**（姓名、国家、状态、日期等）：支持多列（含双列一行、三列摘要栅格），**每格内**仍为上下 KV。  
2. **长字段**（SKU、审批人、备注、规格描述等）：宜**单列独占一块**（或独占一列宽），避免挤压可读性。  
3. **查看类载体**：页面或抽屉内键值区均为**上下 KV**，间距 **§2**；含表或主从的复合关系 **§3**。

---

## 6 一级分段与二级卡片（只读）

只读详情中若使用**一级标题行**、**段间分割线**、**灰底二级区**，量化规则与 **`form-standard.md` §2.4** 完全一致（装饰条 **2×16**、标题 **16 Semibold**、分割线 **1px`--color-border-default`** 等）。  
**二级灰底卡片**背景为 **`--color-bg-page`**，内边距 **24px**、无圆角、无阴影（与子层级编辑区视觉一致时，注意内容须为**只读**）。卡片内嵌表格的表头背景规则见 **`form-standard.md` 第 12 节**脚注（灰底上表头用白底等）。
